## 🐛 Bug Reports

For normal bugs, unexpected behaviour, broken pages, documentation errors, or feature problems, create a GitHub Issue when public discussion is safe.

A useful bug report should include:

- Affected page, feature, or module
- Exact URL
- Expected behaviour
- Actual behaviour
- Steps to reproduce
- Browser and operating system
- Mobile or desktop device
- Screenshots or non-sensitive logs
- Whether login, camera, microphone, or location permission was involved

Do not include passwords, API keys, access tokens, private conversations, payment information, or personal user data.

## 💡 Feature Suggestions

Feature ideas may be submitted through GitHub Issues or through the official contact email.

Please explain:

- The problem the feature would solve
- Intended users

- Expected benefits
- Possible privacy or safety concerns

## 📖 Scripture and Sanskrit Corrections

When reporting a scripture or Sanskrit issue, include:

- Scripture or text name
- Edition or source
- Kanda, chapter, Sarga, Sutra, Sukta, or verse number
- Existing text
- Proposed correction
- Supporting reference
- Known textual variant, when applicable

Do not submit AI-generated Sanskrit as a verified canonical correction without a reliable source.

## 🪔 Panchang and Jyotish Issues

For Panchang or Jyotish calculation problems, include:

- Date
- Local time
- City or location
- Latitude and longitude, when available
- Timezone
- Expected result
- Result displayed by BRAHM-AI
- Reference source used for comparison

Location and timezone details are important for accurate investigation.
